🔥 شرح مفصل عن Zustand

🚀 ما هو Zustand؟

Zustand هو مكتبة إدارة الحالة (State Management) في React، خفيفة وسهلة الاستخدام، وتعتبر بديلاً عن Redux و Context API. تم تطويرها من قبل Poimandres (نفس الفريق الذي طور react-three-fiber).

🛠️ مزايا Zustand مقارنة بـ Redux و Context API

✅ سهولة الاستخدام - يمكنك إنشاء وإدارة الحالة بسطرين أو ثلاثة فقط.
✅ أداء عالٍ - يستخدم React hooks بطريقة تمنع إعادة التصيير غير الضرورية.
✅ لا يحتاج Boilerplate - لا يوجد reducers أو actions أو middleware معقد.
✅ يدعم الاشتراك الجزئي - يمكنك الاشتراك في جزء من الحالة فقط، مما يحسن الأداء.
✅ متعدد الاستخدامات - يعمل في تطبيقات React و React Native بسهولة.

📌 كيفية تثبيت Zustand

قبل البدء، قم بتثبيت Zustand عبر npm أو yarn:

sh
npm install zustand
# أو
yarn add zustand

🏗️ إنشاء متجر Zustand (Store)

في Zustand، نقوم بإنشاء حالة التطبيق باستخدام create وتخزين البيانات داخل store.

📍 مثال بسيط: عداد أرقام

javascript
import { create } from 'zustand'

const useCounterStore = create((set) => ({
  count: 0,
  increase: () => set((state) => ({ count: state.count + 1 })),
  decrease: () => set((state) => ({ count: state.count - 1 })),
}))

🔹 useCounterStore هو hook يسمح لنا باستخدام count وإدارته.
🔹 increase و decrease هما دالتان لتحديث الحالة.

🎯 استخدام Zustand داخل مكون React

javascript
import React from 'react'
import { useCounterStore } from './store'

const Counter = () => {
  const { count, increase, decrease } = useCounterStore()

  return (
    <div>
      <h1>العدد: {count}</h1>
      <button onClick={increase}>+</button>
      <button onClick={decrease}>-</button>
    </div>
  )
}

export default Counter

📌 استخدام اشتراك جزئي (Partial Subscription) لتحسين الأداء

إذا كنت تريد فقط قراءة count بدون الحاجة إلى increase و decrease، يمكنك استخدام اشتراك جزئي:

javascript
const count = useCounterStore((state) => state.count)

📌 بهذه الطريقة، المكون سيعيد التصيير فقط عند تغير count، مما يحسن الأداء.

🔄 حفظ البيانات باستخدام Zustand + LocalStorage

Zustand لا يحتوي على Persist افتراضيًا، ولكن يمكننا استخدام middleware لحفظ البيانات في localStorage:

javascript
import { create } from 'zustand'
import { persist } from 'zustand/middleware'

const useUserStore = create(
  persist(
    (set) => ({
      username: '',
      setUsername: (name) => set({ username: name }),
    }),
    { name: 'user-storage' } // اسم التخزين في localStorage
  )
)

📌 عند إعادة تحميل الصفحة، سيبقى اسم المستخدم محفوظًا.

🔄 استخدام Zustand مع Middleware

يمكنك إضافة middlewares مثل devtools لمراقبة الحالة أثناء التطوير:

javascript
import { create } from 'zustand'
import { devtools } from 'zustand/middleware'

const useStore = create(
  devtools((set) => ({
    count: 0,
    increase: () => set((state) => ({ count: state.count + 1 })),
  }))
)

👀 عند استخدام Redux DevTools Extension، يمكنك رؤية التغييرات بسهولة!

🏆 متى تستخدم Zustand؟

✅ عندما تحتاج إلى إدارة حالة بسيطة بدون تعقيد.
✅ عند الحاجة إلى أداء عالٍ مع إعادة تصيير أقل.
✅ إذا كنت تريد بديلًا أخف وأبسط من Redux.
✅ في React Native حيث يكون الأداء مهمًا جدًا.

⚡ Zustand vs Redux vs Context API

الميزةZustandReduxContext API
سهولة الاستخدام✅ سهل جدًا❌ معقد مع boilerplate✅ سهل
الأداء✅ عالي⚠️ جيد لكن قد يؤدي إلى إعادة تصيير زائدة❌ ضعيف مع حالات كبيرة
حجم المكتبة✅ 1KB فقط❌ أكبر بكثير✅ مدمج في React
اشتراك جزئي✅ مدعوم✅ مدعوم❌ غير مدعوم
التخزين✅ يدعم persist⚠️ يتطلب إعداد إضافي❌ لا يدعم

🎯 الخلاصة

📌 Zustand هو الخيار المثالي إذا كنت تريد مكتبة إدارة حالة خفيفة وسهلة الاستخدام، وأداءً قويًا دون تعقيد Redux. 🚀🔥

الفرق بين Virtual DOM و Shallow Rendering يكمن في المفهوم والاستخدام:

1️⃣ Virtual DOM (الـ DOM الافتراضي)

🔹 المفهوم: هو تقنية تستخدمها React لتحسين الأداء، حيث تحتفظ React بنسخة افتراضية من DOM في الذاكرة، وتقارنها مع النسخة الفعلية (diffing) قبل تحديث الواجهة.
🔹 لماذا؟ لأنه يقلل من عمليات إعادة التصيير (Re-rendering) غير الضرورية، مما يجعل التطبيق أسرع.
🔹 كيف يعمل؟

  1. يتم تحديث Virtual DOM عندما تتغير البيانات.
  2. يقارن React بين النسخة القديمة والجديدة (Diffing).
  3. يتم تحديث DOM الحقيقي فقط في الأجزاء التي تغيرت.

🛠 مثال عملي:
عند تحديث حالة (State) مكون React، React لا يقوم مباشرة بتحديث DOM، بل يحدث Virtual DOM أولًا، ثم يطبق فقط التغييرات المطلوبة.

2️⃣ Shallow Rendering (التصيير السطحي)

🔹 المفهوم: هو أسلوب يُستخدم لاختبار مكونات React عن طريق تصييرها دون تصيير المكونات الفرعية (Child Components).
🔹 لماذا؟ لتبسيط اختبار المكونات وعزلها دون الحاجة إلى التعامل مع DOM الفعلي أو المكونات الداخلية.
🔹 كيف يعمل؟ عند اختبار مكون باستخدام Shallow Rendering، يتم تصيير المستوى الأول فقط من المكون، بينما يتم تجاهل المكونات الفرعية.

🛠 مثال عملي في الاختبارات:

javascript
import { shallow } from 'enzyme';
import MyComponent from './MyComponent';

test('renders correctly', () => {
  const wrapper = shallow(<MyComponent />);
  expect(wrapper.find('h1').text()).toBe('Hello World');
});

🔹 هنا، يتم اختبار MyComponent فقط دون تصيير أي مكونات فرعية.

🔥 الفرق الرئيسي

الميزةVirtual DOMShallow Rendering
المفهومنسخة افتراضية من DOM في الذاكرةطريقة لاختبار المكونات دون تصيير المكونات الفرعية
الاستخدامتحسين الأداء في Reactتسهيل اختبار المكونات الفردية
التأثيريقلل عمليات تحديث الـ DOM الفعلييعزل المكون أثناء الاختبار
أداة مرتبطةReact ReconciliationEnzyme / React Testing Library

🔹 Virtual DOM يتعلق بأداء التطبيق، بينما Shallow Rendering يتعلق باختبار المكونات.

What is controlled and uncontrolled component in React js بالعربي
ChatGPT said:

المكونات المتحكم فيها (Controlled) وغير المتحكم فيها (Uncontrolled) في React.js

في React، هناك طريقتان لإدارة البيانات داخل حقول الإدخال (Forms & Inputs):

  1. المكونات المتحكم فيها (Controlled Components)
  2. المكونات غير المتحكم فيها (Uncontrolled Components)

🟢 1- المكونات المتحكم فيها (Controlled Components)

🔹 هي المكونات التي يتم التحكم في قيمها عبر حالة (State) داخل React.
🔹 أي تغيير في input يتم تخزينه في state، مما يعني أن React تتحكم في القيمة.

مثال على مكون متحكم فيه:

javascript
import React, { useState } from 'react';

const ControlledInput = () => {
  const [text, setText] = useState('');

  return (
    <div>
      <input 
        type="text" 
        value={text} 
        onChange={(e) => setText(e.target.value)} 
      />
      <p>قيمة الإدخال: {text}</p>
    </div>
  );
};

export default ControlledInput;

📌 مميزات المكونات المتحكم فيها:

✅ React تتحكم في البيانات، مما يسهل التحقق من المدخلات (Validation).
✅ يسهل إدارة الأحداث (Events) مثل الضغط على زر.
✅ يمكن ربط البيانات مع Redux أو أي State Management آخر بسهولة.

❌ العيوب:

⚠️ قد يؤدي إلى إعادة التصيير (Re-rendering) بشكل متكرر إذا لم تتم إدارتها بشكل جيد.

🔵 2- المكونات غير المتحكم فيها (Uncontrolled Components)

🔹 يتم تخزين قيمة input داخل DOM نفسه وليس داخل React state.
🔹 يتم استخدام ref للوصول إلى القيم بدلاً من استخدام state.

مثال على مكون غير متحكم فيه:

javascript
import React, { useRef } from 'react';

const UncontrolledInput = () => {
  const inputRef = useRef(null);

  const handleSubmit = () => {
    alert(`القيمة المدخلة: ${inputRef.current.value}`);
  };

  return (
    <div>
      <input type="text" ref={inputRef} />
      <button onClick={handleSubmit}>إرسال</button>
    </div>
  );
};

export default UncontrolledInput;

📌 مميزات المكونات غير المتحكم فيها:

✅ أداء أسرع لأن القيم لا تُخزن في state، مما يقلل إعادة التصيير.
✅ مفيد عند التعامل مع مكتبات خارجية مثل Formik أو Material UI التي تحتاج التحكم المباشر في الحقول.

❌ العيوب:

⚠️ من الصعب التحكم في البيانات وإجراء التحقق (Validation) بسهولة.
⚠️ لا يمكن تحديث الواجهة بشكل مباشر بناءً على تغيير القيم.

🔥 الفرق بين Controlled و Uncontrolled في جدول

الميزةالمكونات المتحكم فيها (Controlled)المكونات غير المتحكم فيها (Uncontrolled)
إدارة البياناتباستخدام state داخل Reactيتم تخزينها في DOM مباشرة
إمكانية التحقق✅ سهل❌ صعب
إعادة التصيير⚠️ أكثر إعادة تصيير✅ أداء أسرع
الوصول إلى القيمباستخدام useStateباستخدام useRef
الاستخدام الأمثلفي النماذج التفاعلية و التحقق من المدخلاتعند التعامل مع مكتبات خارجية أو نماذج كبيرة

🎯 متى تستخدم كل نوع؟

✅ استخدم المكونات المتحكم فيها إذا كنت تحتاج إلى التحكم الكامل في القيم، مثل النماذج المعقدة أو التحقق من المدخلات (Validation).
✅ استخدم المكونات غير المتحكم فيها إذا كنت تريد تحسين الأداء أو تحتاج إلى التعامل مع مكتبات خارجية.

💡 الخلاصة:
🔹 المكونات المتحكم فيها تعتمد على React state وتوفر تحكمًا أفضل.
🔹 المكونات غير المتحكم فيها تعتمد على DOM وتوفر أداءً أسرع ولكن تحكمًا أقل.

🔥 ما هي الـ Hooks في React.js؟

📌 تعريف الـ Hooks

Hooks هي دوال (Functions) في React تسمح لك باستخدام ميزات الحالة (State) ودورة حياة المكونات (Lifecycle Methods) داخل المكونات الوظيفية (Functional Components) بدون الحاجة إلى استخدام المكونات الكلاسيكية (Class Components).

تمت إضافتها في React 16.8 لتحسين إدارة الحالة وإعادة استخدام المنطق بسهولة أكبر.

🎯 لماذا نستخدم الـ Hooks؟

✅ يمكننا إدارة الحالة (State) داخل المكونات الوظيفية.
✅ تسهل إعادة استخدام المنطق في أكثر من مكون بدون الحاجة إلى HOCs أو Render Props.
✅ تقلل من تعقيد الكود مقارنة بالمكونات الكلاسيكية.
✅ تعمل على تحسين أداء التطبيقات عبر تقليل الحاجة إلى إنشاء مكونات إضافية.

🔥 أنواع الـ Hooks الأساسية في React

1️⃣ useState – إدارة الحالة في المكونات الوظيفية

يتيح لنا useState تخزين القيم المتغيرة (State) داخل المكون.

🔹 مثال: زر لزيادة العدد عند الضغط عليه

javascript
import React, { useState } from 'react';

const Counter = () => {
  const [count, setCount] = useState(0); // حالة أولية بقيمة 0

  return (
    <div>
      <p>العدد: {count}</p>
      <button onClick={() => setCount(count + 1)}>زيادة</button>
    </div>
  );
};

export default Counter;

🛠️ كيف يعمل؟

  • useState(0) ينشئ حالة (State) باسم count وقيمتها الأولية 0.
  • setCount(count + 1) يتم استدعاؤه عند الضغط على الزر، مما يحدث إعادة التصيير (Re-render) للمكون بالقيمة الجديدة.

2️⃣ useEffect – تنفيذ الأكواد عند تغييرات معينة

يعمل useEffect كبديل لدوال دورة الحياة مثل componentDidMount و componentDidUpdate و componentWillUnmount في المكونات الكلاسيكية.
يُستخدم لتنفيذ أكواد عند تحميل المكون أو عند تغيير القيم.

🔹 مثال: جلب بيانات من API عند تحميل المكون

javascript
import React, { useState, useEffect } from 'react';

const FetchData = () => {
  const [data, setData] = useState([]);

  useEffect(() => {
    fetch('https://jsonplaceholder.typicode.com/posts')
      .then((response) => response.json())
      .then((json) => setData(json));

    console.log('تم جلب البيانات');
  }, []); // تمرير مصفوفة فارغة يعني أن الكود ينفذ مرة واحدة عند التحميل

  return (
    <div>
      <h2>المنشورات:</h2>
      <ul>
        {data.slice(0, 5).map((post) => (
          <li key={post.id}>{post.title}</li>
        ))}
      </ul>
    </div>
  );
};

export default FetchData;

🛠 كيف يعمل؟

  • useEffect يتم تشغيله مرة واحدة فقط عند تحميل المكون لأننا مررنا مصفوفة فارغة [].
  • يتم استدعاء fetch لجلب البيانات، ثم تحديث data عند وصول البيانات الجديدة.

🔹 حالات استخدام useEffect
✅ جلب البيانات من API.
✅ الاشتراك في WebSocket أو Events.
✅ تنفيذ أكواد عند تحديث الحالة أو الخصائص (props).

3️⃣ useRef – الوصول إلى عناصر DOM بدون إعادة التصيير

🔹 يتم استخدام useRef للحصول على مرجع لعناصر الـ DOM بدون إعادة التصيير عند التغيير.

🔹 مثال: التحكم في إدخال نص بدون useState

javascript
import React, { useRef } from 'react';

const InputFocus = () => {
  const inputRef = useRef(null);

  const handleFocus = () => {
    inputRef.current.focus(); // يجعل التركيز على الحقل
  };

  return (
    <div>
      <input ref={inputRef} type="text" placeholder="اكتب هنا..." />
      <button onClick={handleFocus}>اضغط للتركيز</button>
    </div>
  );
};

export default InputFocus;

🛠 كيف يعمل؟

  • useRef(null) ينشئ مرجعًا للحقل.
  • inputRef.current.focus() يجعل التركيز ينتقل للحقل عند الضغط على الزر.
  • لا يؤدي إلى إعادة التصيير عند التغيير.

4️⃣ useContext – تمرير البيانات بين المكونات بسهولة

🔹 بديل أفضل من props عندما نحتاج إلى تمرير بيانات عبر عدة مكونات متداخلة.

🔹 مثال: مشاركة لون بين مكونات متعددة باستخدام useContext

javascript
import React, { createContext, useContext, useState } from 'react';

// إنشاء الـ Context
const ThemeContext = createContext();

const ThemeProvider = ({ children }) => {
  const [theme, setTheme] = useState('light');
  return (
    <ThemeContext.Provider value={{ theme, setTheme }}>
      {children}
    </ThemeContext.Provider>
  );
};

const ThemedComponent = () => {
  const { theme, setTheme } = useContext(ThemeContext);
  return (
    <div style={{ background: theme === 'light' ? '#fff' : '#333', color: theme === 'light' ? '#000' : '#fff', padding: '20px' }}>
      <p>الثيم الحالي: {theme}</p>
      <button onClick={() => setTheme(theme === 'light' ? 'dark' : 'light')}>تغيير الثيم</button>
    </div>
  );
};

const App = () => (
  <ThemeProvider>
    <ThemedComponent />
  </ThemeProvider>
);

export default App;

🛠 كيف يعمل؟

  • يتم تخزين الثيم في useState وتمريره عبر ThemeContext.Provider.
  • useContext(ThemeContext) يسمح لنا بقراءة البيانات بدون تمرير props لكل مكون.

🔹 useRef و useContext يساعدان في الوصول إلى DOM ومشاركة البيانات بدون إعادة تصيير.

  },

🔥 ما هو Redux؟

🚀 Redux هو مكتبة لإدارة الحالة (State) في تطبيقات JavaScript، خاصة مع React.

يساعد في تنظيم البيانات المشتركة بين المكونات المختلفة، مما يجعل التطبيق أكثر استقرارًا وسهل الصيانة.

🎯 لماذا نحتاج Redux؟

🔹 في تطبيقات React الكبيرة، قد تصبح إدارة البيانات بين المكونات معقدة.
🔹 بدلاً من تمرير props يدويًا بين المكونات، يوفر Redux مخزنًا مركزيًا (Global Store) يمكن لجميع المكونات الوصول إليه.
🔹 يسهل تتبع التغييرات وتحليل الأخطاء بفضل سجل التعديلات (Time Travel Debugging).

🛠️ المفاهيم الأساسية في Redux

Redux يعتمد على 3 مبادئ رئيسية:

1️⃣ 📦 Store (المخزن) – مصدر الحقيقة الوحيد

🔹 Store هو كائن يحتوي على جميع بيانات التطبيق.
🔹 يتم تخزين الحالة (State) في كائن واحد داخل Redux.

🔹 مثال على حالة (State) داخل Store:

js
const initialState = {
  count: 0,
  user: { name: "Ahmed", age: 25 }
};

2️⃣ 🔄 Actions (الأوامر) – كيف نطلب تغييرات؟

🔹 Action هو كائن يخبر Redux ماذا يجب أن يحدث.
🔹 كل Action يحتوي على type يحدد نوع التغيير.

🔹 مثال على Action لزيادة العدد:

js
const incrementAction = { type: "INCREMENT" };

🔹 يمكن إضافة بيانات إضافية (payload) مع Action:

js
const setUserAction = {
  type: "SET_USER",
  payload: { name: "Ali", age: 30 }
};

3️⃣ ⚙️ Reducers (المعالج) – كيف يتم تغيير البيانات؟

🔹 Reducer هو دالة (Function) تأخذ الحالة الحالية (State) و Action، ثم تعيد حالة جديدة (State جديدة).
🔹 يجب أن يكون Reducer دالة نقية (Pure Function) أي لا تغير الحالة القديمة بل تعيد حالة جديدة.

🔹 مثال على Reducer لتحديث العداد:

js
const counterReducer = (state = { count: 0 }, action) => {
  switch (action.type) {
    case "INCREMENT":
      return { count: state.count + 1 };
    case "DECREMENT":
      return { count: state.count - 1 };
    default:
      return state;
  }
};

🔹 2. إنشاء Store باستخدام Reducer

js
import { createStore } from "redux";

const initialState = { count: 0 };

const counterReducer = (state = initialState, action) => {
  switch (action.type) {
    case "INCREMENT":
      return { count: state.count + 1 };
    case "DECREMENT":
      return { count: state.count - 1 };
    default:
      return state;
  }
};

// إنشاء Store
const store = createStore(counterReducer);

export default store;

js
import React from "react";

import Counter from "./Counter";

ReactDOM.render(
  <Provider store={store}>
    <Counter />
  </Provider>,
  document.getElementById("root")
);

🔹 4. استخدام Redux داخل مكون React

js
import React from "react";
import { useSelector, useDispatch } from "react-redux";

const Counter = () => {
  const count = useSelector((state) => state.count);

      <button onClick={() => dispatch({ type: "DECREMENT" })}>-</button>
    </div>
  );
};

export default Counter;
